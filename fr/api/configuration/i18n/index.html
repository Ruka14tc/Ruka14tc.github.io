i18n

Un guide pour configurer l'internationalisation (i18n) de votre shadcn-nuxt-docs

Configuration de l'internationalisation

Ajouter la configuration i18n

Tout d'abord, vous devez ajouter la configuration i18n votre projet Nuxt.js. Ouvrez votre fichier nuxt.config.ts et ajoutez la configuration suivante :

export default defineNuxtConfig({
  // Autres configurations nuxt...
  i18n: {
    defaultLocale: 'en',
    locales: [
      {
        code: 'en',
        name: 'English',
        language: 'en-US',
      },
      {
        code: 'xx', // code de langue  2 lettres (e.g. 'fr')
        name: 'Xxxxx', // nom de la langue (e.g. 'Franais')
        language: 'xx-XX', // code ISO de la langue (e.g. 'fr-FR')
      },
    ],
  },
});

Organiser vos fichiers de contenu

Pour prendre en charge plusieurs langues, vous devez organiser vos fichiers de contenu selon une structure de rpertoire spcifique. Suivez cette arborescence de fichiers :

  • content/
    • index.md
    • xx/
      • index.md
  • Le rpertoire content contient vos fichiers de contenu dans la langue par dfaut.
  • Le rpertoire xx l'intrieur de content contient les traductions de vos fichiers de contenu, xx tant le code de la langue (par exemple, fr/index.md pour le franais).
Note

Si vous utilisez des codes de langue qui ne sont pas deux lettres (par exemple, zh-CN et en-US), le nom de votre dossier de contenu doit tre en minuscules.

Alternativement, vous pouvez dfinir content.respectPathCase: true dans nuxt.config.ts pour utiliser la mme casse que le code de langue.

nuxt.config.ts
export default defineNuxtConfig({
  content: {
    respectPathCase: true,
  },
});

Crer i18n/i18n.config.ts

Ensuite, crez un nouveau fichier nomm i18n/i18n.config.ts. Ce fichier contiendra la configuration de vos messages i18n traduire. Ajoutez le code suivant au fichier :

import { defineI18nConfig } from '@nuxtjs/i18n';

export default defineI18nConfig(() => ({
  legacy: false,
  locale: 'en',
  messages: {
    en: {
      // Ajoutez ici toutes les chanes anglaises dfinies dans app.config.ts, par exemple les titres dans header.nav ou toc.title
      // Les chanes d'interface utilisateur par dfaut sont dj traduites prtes  l'emploi

      shiki: 'shiki', // par exemple : titre du site
    },
    xx: {
      // Ajoutez ici toutes les traductions

      shiki: '', // par exemple : titre du site
    },
  },
}));

Traduction de l'interface utilisateur

La langue de l'interface utilisateur suivra la langue slectionne dans le slecteur de langue. Les traductions d'interface sont fournies prtes l'emploi pour les langues suivantes :

  • en (English)
  • fr (Franais)
  • zh-CN ()
  • zh-TW ()
  • km ()
  • ja ()
  • ru ()
  • ko ()
  • hi ()
  • bn ()

Si vous souhaitez ajouter d'autres langues, n'hsitez pas contribuer au projet.

Si vous souhaitez ajouter vos propres traductions, vous pouvez le faire en suivant la structure dans le fichier i18n/i18n.config.ts. Ajoutez simplement les traductions pour les cls que vous souhaitez traduire dans l'objet de langue respectif.
